## 一键脚本

当前目录已生成 skill 脚本：

```bash
bash skills/apk-resign/scripts/resign_apk.sh app-channela-release.apk
```

脚本默认会：

1. 检查 `apktool`、`keytool`、`sdkmanager`、Android build-tools。
2. 缺少 `apktool` 时通过 Homebrew 安装。
3. 缺少 build-tools 时通过 `sdkmanager` 安装 `build-tools;36.0.0`。
4. 用工作区内 `apktool-framework/` 解析 APK，避免写入 `~/Library/apktool/framework`。
5. 使用 `apktool b` 重新打包。
6. 生成或复用 `dx-apk-resign.jks`。
7. 使用 `zipalign -p -f -v 4` 对齐。
8. 使用 `apksigner sign` 签名。
9. 使用 `apksigner verify --verbose --print-certs` 验证。

## 手工命令

如果需要手工执行，可按下面顺序：

```bash
apktool d -f app-channela-release.apk -o app-channela-release -p apktool-framework
apktool b app-channela-release -p apktool-framework
keytool -genkeypair -v \
  -keystore dx-apk-resign.jks \
  -storepass dxapk123456 \
  -keypass dxapk123456 \
  -alias dx-apk-resign \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -dname "CN=dx-apk-resign, OU=dev, O=dx, L=Shanghai, ST=Shanghai, C=CN"
/opt/homebrew/share/android-commandlinetools/build-tools/36.0.0/zipalign \
  -p -f -v 4 \
  app-channela-release/dist/app-channela-release.apk \
  app-channela-release-rebuilt-aligned.apk
/opt/homebrew/share/android-commandlinetools/build-tools/36.0.0/apksigner sign \
  --ks dx-apk-resign.jks \
  --ks-key-alias dx-apk-resign \
  --ks-pass pass:dxapk123456 \
  --key-pass pass:dxapk123456 \
  --out app-channela-release-rebuilt-signed.apk \
  app-channela-release-rebuilt-aligned.apk
/opt/homebrew/share/android-commandlinetools/build-tools/36.0.0/apksigner verify \
  --verbose --print-certs \
  app-channela-release-rebuilt-signed.apk
```

## 注意事项

- 重签名证书与原 APK 官方证书不同，不能覆盖安装原官方签名包；设备上如已安装原包，需要先卸载。
- 如果要覆盖安装，必须使用原始签名证书重新签名。
- `targetSdkVersion=36` 的 APK 不建议只用 `jarsigner`，应使用 `apksigner` 的 APK Signature Scheme v2/v3。
- Android command line tools 默认安装在 `/opt/homebrew/share/android-commandlinetools`。
- `sdkmanager` 需要 JDK 17+；本机可使用 `/opt/homebrew/opt/openjdk/libexec/openjdk.jdk/Contents/Home`。
